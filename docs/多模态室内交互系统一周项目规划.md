# 多模态室内交互系统五天开发计划

## 1. 开发目标

五天内完成以下开发工作：

- 基于 `YOLO` 的目标检测与指向目标定位
- 基于 `DeepSeek VLM` 的视觉理解、视觉问答与最终回答生成
- 基于规则的路由器，用于验证三类链路
- 基于工具调用的最小 Agent，用于替代规则路由

开发场景固定为实验室。

## 2. 开发链路

### 2.1 纯文本链路

`ASR -> DeepSeek VLM(纯文本模式) -> TTS`

### 2.2 视觉直答链路

`ASR + image/crop -> DeepSeek VLM -> TTS`

### 2.3 指向增强链路

`hand_stream -> YOLO -> crop/上下文 -> DeepSeek VLM -> TTS`

## 3. 模块开发任务

### 3.1 感知模块

当前已有基础代码，文件位置：`services/hand_stream.py`

本阶段开发任务：

- 确认实验室目标物清单
- 基于 `services/hand_stream.py` 调整 `YOLO` 检测输出
- 调整指向命中结果输出格式
- 提供 bbox 和 crop 接口
- 输出统一结构化视觉上下文

建议输出结构：

```json
{
  "timestamp": 0,
  "has_hand": true,
  "pointing": true,
  "target_class": "cup",
  "bbox": [120, 80, 300, 260],
  "confidence": 0.81,
  "frame_path": "rviz_captured_images/latest.jpg"
}
```

### 3.2 DeepSeek VLM 模块

开发任务：

- 接入 `DeepSeek VLM` 接口
- 验证纯文本调用
- 验证整图调用
- 验证基于 bbox/crop 的局部图像调用
- 统一视觉问题输入格式与返回格式

### 3.3 路由模块

开发任务：

- 实现规则路由器
- 判断是否需要视觉输入
- 判断是否需要指向上下文
- 判断是否直接调用 `DeepSeek VLM`
- 打通三类链路的调度逻辑

### 3.4 Agent 模块

开发任务：

- 将 `YOLO` 与 `DeepSeek VLM` 封装为工具
- 实现最小 Agent 工具选择流程
- 对比 Agent 与规则路由的执行结果

## 4. 人员分工

### 4.1 物品识别与感知负责人 1人

负责内容：

- 实验室场景确认
- 目标物清单确认
- 基于 `services/hand_stream.py` 调整 `YOLO` 检测与 `hand_stream` 输出
- 指向命中稳定性验证
- bbox 和 crop 输出接口

交付内容：

- 实验室场景样例数据
- 结构化感知输出
- 指向识别测试结果

### 4.2 路由器与 Agent 负责人 1到2人

负责内容：

- 规则路由器设计与实现
- `DeepSeek VLM` 接入
- 视觉上下文与 `DeepSeek VLM` 提示拼接整合
- Agent 工具封装与联调

交付内容：

- 路由器版本
- 最小 Agent 版本
- 三链路联调结果

## 5. 五天开发安排

### 第 1 天

- 基于 `services/hand_stream.py` 调整感知模块输出结构
- `DeepSeek VLM` 接口接入与单独调用验证
- 路由模块完成基础骨架

### 第 2 天

- 验证 `YOLO -> 指向上下文` 输出稳定
- 验证 `DeepSeek VLM` 单独调用可用
- 打通简单视觉问题直答链路

### 第 3 天

- 打通 `YOLO -> DeepSeek VLM` 主链路
- 完成“指向某物并提问”的完整回答闭环

### 第 4 天

- 完成三类问题规则路由验证
- 修正输入格式、输出格式与链路异常处理

### 第 5 天

- 接入最小 Agent
- 验证工具选择流程
- 与规则路由结果进行对比
- 系统联调
- 指标整理

## 6. 验收与测试

重点验证以下内容：

- 指向目标识别成功率
- 简单视觉问题回答正确率
- 指向场景下回答相关性
- 规则路由成功率
- Agent 工具选择成功率

建议测试样例：

- 20 个视觉问答样例
- 20 个指向识别样例

## 7. 交付内容

- 可运行的多模态 demo
- 规则路由器版本
- 最小 Agent 版本
- 测试样例与指标统计表
